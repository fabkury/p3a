API

Enter your Giphy API key. Giphy channels will not work without a valid key. You can create a key for free at https://developers.giphy.com/. To remove a stored key, clear the field and save.
By default, Giphy keys are beta-level and limited to 100 requests per hour. If you have many Giphy channels on the same playset you might run into 429 errors (too many requests). Each Giphy query (request) returns up to 50 artworks, so a 500-artwork channel takes 10 requests to refresh.

Museums

Channels that play artwork from the Art Institute of Chicago, the Rijksmuseum, the Victoria and Albert Museum, the Wellcome Collection, the Statens Museum for Kunst (SMK), and Harvard Art Museums via IIIF. Build a museum channel from Playsets > Add Channel > Museum.

Refresh interval controls how often the device re-queries each museum's listing API for new artwork. Cache size caps the entries kept per channel (32-4096).

Rate limits: The Art Institute of Chicago publishes a hard limit of 60 requests per minute per IP. A single AIC channel refresh issues up to ~11 paginated requests at the default 1024-entry cache size (or ~41 at the 4096 maximum), so several AIC channels refreshing simultaneously can trigger throttling. Harvard Art Museums caps at 2 500 requests per day per API key, and Smithsonian (api.data.gov) at 1 000 requests per hour per key. The four other museums (Rijksmuseum, V&A, Wellcome Collection, SMK) don't publish a limit; p3a applies a 60-second defensive cooldown if any museum returns a 429. The browse previewer and refresh dispatcher both honor the shared cooldown when a 429 is observed.

Harvard Art Museums (API key)

Enter your Harvard Art Museums API key. HAM channels will not refresh without a valid key, and the museum browser shows a prompt to enter one when you click into HAM until a key is saved. You can request a free key at harvardartmuseums.org/collections/api. The key counts against a per-key quota of 2 500 requests per day; one HAM channel refresh costs ~11 calls at the default 1 024-entry cache size.

Smithsonian (API key)

Enter your api.data.gov API key. Smithsonian channels will not refresh without a valid key, and the museum browser shows a prompt to enter one when you click into Smithsonian until a key is saved. Get a free key at api.data.gov/signup/ — the same key works for any api.data.gov service. The default registered-key quota is 1 000 requests per hour; one Smithsonian channel refresh costs ~11 calls at the default 1 024-entry cache size. (Do not use DEMO_KEY — it is capped at ~30 requests per hour per IP and will throttle mid-refresh.)
